# 函数基础（Function Basics）— 续

---

## 本地函数（Local Functions）

本地函数是指 **定义在另一个函数内部的函数（functions declared inside other functions）**，也称为嵌套函数（nested functions）。它们可以访问外部函数的变量，实现代码的封装和复用。

> "Kotlin supports local functions, which are functions inside other functions. A local function can access local variables of outer functions (the closure)." — Kotlin 官方文档

### 基本语法

```kotlin
fun outerFunction() {
    // 本地函数定义
    fun localFunction() {
        println("I'm a local function!")
    }
    
    // 调用本地函数
    localFunction()
}
```

### 函数内定义函数（Functions Inside Functions）

本地函数的主要用途是 **提取重复逻辑而不污染外部作用域**：

```kotlin
// ❌ 不使用本地函数 — 验证逻辑重复
fun saveUser(user: User) {
    if (user.name.isEmpty()) {
        throw IllegalArgumentException("Name cannot be empty")
    }
    if (user.email.isEmpty()) {
        throw IllegalArgumentException("Email cannot be empty")
    }
    if (user.age < 0) {
        throw IllegalArgumentException("Age cannot be negative")
    }
    // 保存用户...
}

// ✅ 使用本地函数 — 提取重复的验证逻辑
fun saveUser(user: User) {
    fun validate(value: Any?, fieldName: String) {
        when {
            value == null -> throw IllegalArgumentException("$fieldName cannot be null")
            value is String && value.isEmpty() -> 
                throw IllegalArgumentException("$fieldName cannot be empty")
            value is Int && value < 0 -> 
                throw IllegalArgumentException("$fieldName cannot be negative")
        }
    }
    
    validate(user.name, "Name")
    validate(user.email, "Email")
    validate(user.age, "Age")
    // 保存用户...
}
```

### 闭包访问（Closure Access）

本地函数可以访问外部函数的 **所有变量和参数**，这称为闭包（closure）：

```kotlin
fun processNumbers(numbers: List<Int>, threshold: Int) {
    var count = 0  // 外部变量
    
    // 本地函数可以访问 numbers, threshold, count
    fun isValid(n: Int): Boolean {
        return n > threshold  // 访问外部参数
    }
    
    fun process(n: Int) {
        if (isValid(n)) {
            count++  // 修改外部变量
            println("Processing: $n")
        }
    }
    
    numbers.forEach { process(it) }
    println("Processed $count numbers above $threshold")
}

processNumbers(listOf(1, 5, 10, 15, 20), 8)
// Processing: 10
// Processing: 15
// Processing: 20
// Processed 3 numbers above 8
```

### 利用闭包简化代码

由于闭包特性，本地函数可以直接访问外部参数，无需重复传递：

```kotlin
// ❌ 不使用本地函数 — 需要传递 user 参数
fun saveUser(user: User) {
    validate(user, user.name, "Name")   // 重复传递 user
    validate(user, user.email, "Email")
    // ...
}

private fun validate(user: User, value: String, field: String) { /*...*/ }

// ✅ 使用本地函数 — 直接访问 user
fun saveUser(user: User) {
    fun validate(value: String, fieldName: String) {
        if (value.isEmpty()) {
            throw IllegalArgumentException(
                "Cannot save user ${user.id}: $fieldName is empty"  // 直接访问 user
            )
        }
    }
    
    validate(user.name, "Name")    // 无需重复传递 user
    validate(user.email, "Email")
    // 保存用户...
}
```

### 作用域限制（Scope Restrictions）

本地函数 **只在定义它的函数内部可见**，这是一种有效的封装手段：

```kotlin
fun publicApi() {
    // 本地函数 — 实现细节被隐藏
    fun helper1() { /*...*/ }
    fun helper2() { /*...*/ }
    
    helper1()
    helper2()
}

// helper1()  // ❌ 编译错误：Unresolved reference
// helper2()  // ❌ 编译错误：无法在外部访问
```

### 多层嵌套（Nested Local Functions）

本地函数可以继续嵌套，但要注意可读性：

```kotlin
fun outer() {
    fun middle() {
        fun inner() {
            println("Deepest level")
        }
        inner()
    }
    middle()
}

// ⚠️ 过深的嵌套会降低可读性，一般不建议超过两层
```

### 递归本地函数

本地函数也可以递归调用自身：

```kotlin
fun calculateFactorial(n: Int): Long {
    fun factorial(num: Int): Long {
        return if (num <= 1) 1L else num * factorial(num - 1)
    }
    
    require(n >= 0) { "Number must be non-negative" }
    return factorial(n)
}

println(calculateFactorial(5))  // 120
```

### 常见陷阱提醒（Common Pitfalls）

```kotlin
// ⚠️ 陷阱 1: 默认参数 + 重载 可能导致歧义
fun test(a: Int, b: Int = 0) = "v1"
fun test(a: Int) = "v2"
// test(5)  // 编译错误：Ambiguous call

// ⚠️ 陷阱 2: vararg 位置不当导致必须用命名参数
fun log(vararg items: String, level: String) { }
log("a", "b", level = "INFO")  // level 必须命名

// ⚠️ 陷阱 3: 中缀函数优先级
1 + 2 shl 3  // = (1 + 2) shl 3 = 24，不是 1 + (2 shl 3)

// ⚠️ 陷阱 4: 本地函数必须先定义后使用
fun outer() {
    // inner()  // ❌ 错误：还未定义
    fun inner() { }
    inner()  // ✅ 正确
}
```

## 📝 练习题

### 题目 1

以下关于本地函数的说法，**错误** 的是？

A. 本地函数可以访问外部函数的参数和局部变量
B. 本地函数可以修改外部函数中用 `var` 声明的变量
C. 本地函数可以在定义之前被调用
D. 本地函数对外部函数以外的代码不可见

【答案】C

【解析】本地函数 **必须先定义后使用**（must be defined before use），不能在定义之前调用，这与 JavaScript 的函数提升（hoisting）不同。选项 A 正确，本地函数可以访问外部的所有变量（闭包特性）；选项 B 正确，闭包可以修改 `var` 变量；选项 D 正确，本地函数的作用域仅限于定义它的函数内部。

### 题目 2

以下代码的输出是什么？

```kotlin
fun counter(): () -> Int {
    var count = 0
    fun increment(): Int {
        count++
        return count
    }
    return ::increment
}

fun main() {
    val c1 = counter()
    val c2 = counter()
    println("${c1()} ${c1()} ${c2()}")
}
```

A. `1 2 3`
B. `1 2 1`
C. `1 1 1`
D. 编译错误

【答案】B

【解析】每次调用 `counter()` 都会创建一个新的 `count` 变量和新的 `increment` 函数，它们形成独立的闭包（closure）。`c1` 和 `c2` 是两个独立的闭包实例，各自维护自己的 `count`。所以 `c1()` 第一次返回 1，第二次返回 2；而 `c2()` 有自己的 `count`，从 0 开始，第一次调用返回 1。这展示了闭包捕获变量的特性。
